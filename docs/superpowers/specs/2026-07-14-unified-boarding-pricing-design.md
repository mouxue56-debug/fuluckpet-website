# 猫犬小动物统一寄养价格设计

日期：2026-07-14

## 目标

把猫、犬、兔子等小动物的寄养价格收敛为一套简单、可解释、可计算的公开规则，并让公开页面、估价器、结构化数据、聊天知识和员工看到的说明始终使用同一价源。

本轮同时取消会员价和日期加价。福楽卒業猫保留真正有辨识度的专属待遇：寄养和洗护均为 30%OFF，且与长住优惠不叠加。

犬服务仍保持 `現在受付停止`，只更新准备中预计价和只读估价，不开放预约、LINE 接单、犬 Service/Offer Schema 或犬日历写入。

## 公开基础价

所有金额均为含税参考价。

| 动物 | 公开分类 | 基础价 |
|---|---|---:|
| 猫 | 不按体重细分 | ¥4,000 / 1泊 |
| 小型犬 | 10kg 未满 | ¥5,000 / 1泊 |
| 中型犬 | 10kg 以上、20kg 未满 | ¥5,500 / 1泊 |
| 大型犬 | 20kg 以上 | ¥6,500 / 1泊 |
| 兔子等 | 现行 60cm 笼位标准 | ¥1,500 / 1泊 |
| 仓鼠等 | 小动物笼位单位 | ¥500 / 1泊 |

犬体重分档采用大阪公开服务中常见且最容易解释的 `<10kg / 10–20kg / >=20kg`。现场仍可因体型、行为、笼位或照护条件拒绝接待或要求事前确认，但不得在估价器中偷偷改价。

## 统一长住规则

猫、犬、兔子和仓鼠等全部使用同一套阶梯：

| 住宿晚数 | 适用率 | 等价优惠 |
|---|---:|---:|
| 1–6 晚 | 100% | 无 |
| 7–13 晚 | 95% | 5%OFF |
| 14–20 晚 | 90% | 10%OFF |
| 21–29 晚 | 85% | 15%OFF |
| 30 晚以上 | 80% | 20%OFF |

阶梯按整次住宿的晚数判断，不按每周分段累计。普通寄养金额为 `基础价 × 晚数 × 适用率`，最终合计按 `Math.round(金额 / 100) × 100` 四舍五入到 ¥100。页面优先展示“合计优惠率”，不展示容易产生误解的虚假精确单晚价。

示例：普通猫住 30 晚，合计为 `¥4,000 × 30 × 80% = ¥96,000`；小型犬住 30 晚，准备中参考合计为 `¥5,000 × 30 × 80% = ¥120,000`。

## 毕业猫规则

福楽卒業猫的寄养和洗护统一为 70%：

- 寄养：`¥4,000 × 晚数 × 70%`。
- 短毛洗护套餐：`¥4,000 × 70% = ¥2,800`。
- 长毛洗护套餐：`¥6,000 × 70% = ¥4,200`。
- 可优惠洗护单项同样使用 70%；现行明确不可优惠或需要现场报价的项目继续按原规则处理。
- 毕业猫 70% 与长住优惠不相乘、不相加；系统只采用更有利的毕业猫 70%。
- 估价结果必须明确显示 `福楽卒業猫 30%OFF（他の割引と併用不可）`。

## 删除的价格概念

本轮从产品和公开合同中删除以下概念：

- 会员价、会员复选框、会员折扣说明；所有客人默认使用同一公开基础价。
- 周末、祝日、学校假期、黄金周、盂兰盆节和年末年始加价。
- 猫寄养旧 15% 卒業猫折扣；由统一 30%OFF 取代。
- 猫、犬和小动物彼此不同的长住阶梯。

日期和日历数据仍可用于营业日、入住/退房和可预约性判断，但不得再参与价格计算。不得保留值为 `1` 或 `0` 的伪会员价、伪加价字段作为公开价格合同的一部分；如旧消费者需要过渡，只能在边界适配器中明确标注兼容用途并有删除测试。

## 单一价源与数据流

`boarding-public-config.js` 是价格与折扣政策的唯一人工维护入口：

1. 配置定义基础价、犬体重边界、统一长住阶梯和毕业猫 70%。
2. `boarding-public-calc.js` 只负责纯计算，不包含手写价格。
3. `dog-services-projection.js` 从同一配置生成犬准备中投影，并继续携带 `preparing:true / accepting:false`。
4. boarding、grooming 和 estimate 页面从配置或静态生成器渲染，不复制数字。
5. JSON-LD 只发布当前允许公开接单的猫与小动物价格；犬 gate 为 false 时继续不发布犬 Offer。
6. 聊天知识和员工可见价目由静态价目生成器从同一配置投影。

生成器必须失败关闭：价源缺字段、阶梯不连续、犬体重边界重叠、未知优惠或毕业猫折扣不是 70% 时，停止生成，不保留一半新价、一半旧价的页面。

## 页面与估价器

### 寄养页

- 猫主价更新为 `¥4,000 / 1泊`。
- 长住说明改成统一的 5%/10%/15%/20% 阶梯。
- 显示毕业猫寄养 30%OFF，明确不可叠加。
- 小动物卡片保留现行基础价，但改用统一长住说明。
- 犬准备中区显示体重边界和 `¥5,000 / ¥5,500 / ¥6,500`，继续突出 `現在受付停止` 与針中野予定。
- 删除会员、日期加价和旺季说明。

### 洗护页

- 猫洗护基础套餐和现行单项价格不变。
- 毕业猫 30%OFF 文案扩展为“寄养・洗护”，但洗护计算仍使用护理价目。
- 犬准备中护理价不因本轮寄养调整而改变。

### 估价器

- 删除会员复选框和所有相关结果文案。
- 猫选择毕业猫时使用 70%，并隐藏或明确覆盖长住优惠，结果不得显示两个折扣。
- 普通猫、犬、小动物都显示同一阶梯名称和最终合计。
- 删除每晚日期加价明细。
- 犬仍可只读试算，但结果没有预约、LINE 或日历写入动作。

## 兼容与迁移

这是静态配置和纯计算逻辑迁移，不修改正式客户、预约或动物数据，不需要数据库迁移。

旧 URL、页面路径和犬 `public:false` gate 保持不变。若浏览器缓存旧脚本，生成流程必须统一提升相关资源版本，使 HTML、配置、计算器和估价器同批更新。

## 测试与验收

实现必须采用测试先行：

1. 先写失败测试，锁定基础价、犬体重边界、统一阶梯、毕业猫 70% 和不叠加规则。
2. 锁定不存在会员折扣、日期加价和旧阶梯字段。
3. 验证页面、Schema、聊天知识和估价器均不包含旧价 `¥4,800 / ¥7,400 / ¥8,200 / ¥8,900` 或旧优惠文案。
4. 验证犬 gate 仍为 `public:false`，无犬预约 CTA、无犬 Offer Schema、无犬日历写入。
5. 验证 1、6、7、13、14、20、21、29、30 晚边界，以及毕业猫短住和长住。
6. 运行生成器两次并通过 generated clean 校验、全量 Node 测试和 Worker 安全检查。
7. 本地浏览器检查寄养、洗护、估价器的桌面和手机页面，确认价格层级、体重说明、停止接单状态、空/错误状态和控制台。
8. 发布后分别确认 Git/CI、Pages 静态输出和 Worker 版本；只有 Worker 代码或聊天知识实际改变时才部署 Worker。

## 明确不做

- 不开放犬接单，不改变犬 gate。
- 不修改猫洗护基础套餐或犬护理单项价格。
- 不修改兔子、仓鼠等基础价；只统一长住算法。
- 不增加犬一级菜单。
- 不引入会员账号或优惠码系统。
- 不恢复节假日或旺季加价。
- 不搬运 STORES 商品，不改变支付流程。
