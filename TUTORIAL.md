## 1. 整体思路：从零到上线

### 为什么用静态 HTML，不用 WordPress？

| | 静态 HTML (我们的方案) | WordPress |
|---|---|---|
| **费用** | 完全免费（GitHub Pages 免费托管） | 需要付费服务器（每月 $5-20+） |
| **速度** | 极快（只是读取文件） | 较慢（每次访问要查数据库） |
| **维护** | 基本不需要维护 | 需要经常更新、防黑客 |
| **安全性** | 非常安全（没有数据库可以攻击） | 容易被黑（全球最多被攻击的系统） |
| **自由度** | 100% 自定义 | 受主题和插件限制 |

**简单说**：静态网站就像一张印好的名片——简单、快速、不会坏。WordPress 像一台复杂的机器——功能多但需要维护。

### 为什么用 GitHub Pages？

- **完全免费** — 不花一分钱
- **自带 HTTPS** — 网址有小锁，浏览器不会报"不安全"
- **支持自定义域名** — 可以用 fuluckpet.com
- **自动部署** — 推送代码后 1-2 分钟自动上线
- **全球 CDN** — 世界各地访问都快

### 为什么用 Cloudflare？

- **免费 CDN** — 你的网站内容被缓存到全球服务器，日本的客户访问更快
- **免费 SSL** — 自动配置 HTTPS 加密
- **DNS 管理** — 管理域名指向哪里
- **域名注册** — 可以直接在 Cloudflare 买域名，价格透明无加价

### 整体架构（三角形）

```
      你的电脑
    (写代码修改)
        |
        | git push（上传）
        ↓
   GitHub Pages
   (存放代码 + 托管网站)
        |
        | 当有人访问 fuluckpet.com
        ↓
    Cloudflare
   (域名 + CDN + 安全防护)
        |
        | 把网页返回给访客
        ↓
    访客的浏览器
   (看到你的网站)
```

### 三种文件类型

把网站想象成一栋房子：

| 文件类型 | 比喻 | 作用 | 我们的文件 |
|----------|------|------|------------|
| **HTML** | 房子的结构（墙壁、房间） | 网页的内容 — 文字、图片、按钮 | `index.html` 等 8 个页面 |
| **CSS** | 装修（颜色、布局、字体） | 网页的样式 — 好不好看 | `style.css` |
| **JS** | 电器（开关、电梯） | 网页的交互 — 点击会怎样 | `script.js`, `i18n.js` |

```

2. 浏览器下载 index.html
3. HTML 里写着"我还需要 style.css 和 script.js"
4. 浏览器继续下载这两个文件
5. 三个文件组合在一起 → 你看到的漂亮网页
```

## 3. 域名和 DNS 原理

### 域名 = 你在互联网上的门牌号

- `fuluckpet.com` 就是你的门牌号
- 没有域名的话，访客只能输入一串数字（IP 地址），像 `185.199.108.153`
- 域名让人更容易记住你的网站

### DNS = 互联网的电话簿

当有人输入 `fuluckpet.com`：

```
1. 浏览器问 DNS：fuluckpet.com 在哪里？
2. DNS 回答：在 185.199.108.153（GitHub Pages 的服务器）
3. 浏览器去那个地址拿网页
```

DNS 就像电话簿 — 你告诉它名字（fuluckpet.com），它告诉你号码（IP 地址）。

### CNAME 文件 = 告诉 GitHub"这是我的域名"

项目里有个 `CNAME` 文件，内容就一行：`fuluckpet.com`
它的作用是告诉 GitHub Pages："当有人访问 fuluckpet.com 时，请显示这个仓库的网页。"

### Cloudflare = 你的"保安 + 快递站"

Cloudflare 坐在访客和 GitHub 之间：
- **快递站（CDN）**：把你的网页缓存到全球各地的服务器，日本客户不用跑到美国去拿
- **保安（安全）**：拦截恶意攻击，保护你的网站
- **门牌管理（DNS）**：管理 fuluckpet.com 指向哪里

## 4. SEO 搜索引擎优化

SEO = 让 Google 能找到你的网站，并且排名靠前。

### 我们做了什么？

#### sitemap.xml — "Google，这是我所有的页面"
```
就像给 Google 一份地图，告诉它：
"我的网站有这 7 个页面，请都来看看。"
```

#### robots.txt — "Google，这些地方别看"
```
告诉 Google 爬虫：
"你可以看所有页面，但别看 /admin/ 和 /api/，那是后台。"
```

#### Google Search Console — "Google，请收录我的网站"
```
相当于去 Google 那里"报到注册"。
告诉 Google："我有个新网站，这是地址，请来抓取。"
```

#### Google Analytics 4 (GA4) — "来了多少人？"
```
一个统计工具：
- 每天有多少人访问？
- 他们从哪里来？（Google 搜索？Instagram？）
- 看了哪些页面？
- 停留了多久？
```

#### meta 标签 — Google 搜索结果里显示什么
```html
<title>サイベリアン｜大阪・福楽キャッテリー</title>
<meta name="description" content="大阪的西伯利亚猫专门猫舍...">
```
这两行决定了 Google 搜索结果里显示的标题和描述。

#### OGP 标签 — LINE/Twitter 分享预览
当有人把你的网站链接发到 LINE 或 Twitter 时，会显示一张预览图 + 标题 + 描述。
OGP 标签就是控制这个预览长什么样的。

#### JSON-LD — 帮 Google 理解你的生意
```
告诉 Google：
"这是一个宠物店/猫舍，在大阪，老板是谁，电话是多少。"
Google 可能会在搜索结果里显示更丰富的信息。
```

#### Google 收录需要时间
- 提交 sitemap 后，Google 不会马上显示你的网站
- 首页：几天到 1-2 周
- 其他页面：1-4 周
- 排名稳定：1-3 个月
- 这是正常的，耐心等待

## 5. 管理面板工作原理

### localStorage = 你浏览器里的"小笔记本"

管理面板的数据存在你的浏览器里，就像浏览器自带了一个小笔记本：

```
浏览器笔记本 (localStorage)
├── fuluckData = {你添加的所有猫咪信息}
└── fuluckPassword = "fuluck2025"
```

**重要理解**：
- 数据只在**你当前用的这个浏览器**里
- 换了电脑 → 数据不在了
- 换了浏览器（比如从 Chrome 换到 Safari）→ 数据不在了
- 清除浏览器缓存 → 数据没了

这就是为什么**定期备份（导出 JSON）非常重要**！

### HTML 导出 = 把后台数据变成网页代码

管理面板里的"HTML 导出"功能是这样工作的：

```
1. 你在后台添加了一只新猫的信息
2. 点击"生成 HTML"
3. 后台根据你填的信息，自动生成一段 HTML 代码
4. 你把这段代码复制，粘贴到 kittens.html 里
5. git push 上传到 GitHub
6. 网站上就显示新猫了
```

### Google Photos 链接 = 不占空间的照片方案

为什么用 Google Photos 链接而不是直接上传图片？
- GitHub Pages 空间有限
- Google Photos 空间大且免费
- 你本来就要用 Google Photos 分享照片给客户
- 只需要粘贴链接，简单方便

## 6. 多语言 (i18n) 原理

i18n 是 "internationalization" 的缩写（i + 中间18个字母 + n）。

### 工作原理

```
i18n.js 里有一本"词典"：
{
  "hero_title": {
    "ja": "運命の子猫に出会う",      ← 日语
    "en": "Meet Your Destiny",       ← 英语
    "zh": "遇见命中注定的小猫"        ← 中文
  }
}

HTML 里标记了哪些文字需要翻译：
<h1 data-i18n="hero_title">運命の子猫に出会う</h1>

当访客点击"English"时：
script.js 找到所有有 data-i18n 的元素
→ 查词典
→ 把文字换成英语版本
```

**注意**：这是"客户端翻译"，三种语言都在同一个页面里，只是切换显示。不是三个不同的网页。

## 7. Git 版本管理入门

### Git = 你代码的时光机

把 Git 想象成一个超级存档系统：

| 操作 | 比喻 | 命令 |
|------|------|------|
| **git add** | 选择要存档的文件 | `git add index.html` |
| **git commit** | 按下"存档"按钮，写个备注 | `git commit -m "更新了首页"` |
| **git push** | 把存档上传到云端（GitHub） | `git push origin main` |
| **git pull** | 从云端下载最新版本 | `git pull origin main` |

### 为什么 Git 好用？

1. **永远可以回退** — 改坏了？回到上一个存档就行
2. **有完整的修改记录** — 谁在什么时候改了什么，一目了然
3. **自动部署** — push 到 GitHub 后，网站自动更新

### 实际操作流程

```
想修改网站？按这个顺序：

1. git pull          ← 先拿最新版本
2. 修改文件          ← 做你的修改
3. git add 文件名     ← 选择要保存的文件
4. git commit -m "说明" ← 存档并备注
5. git push          ← 上传，网站自动更新
```

## 8. 如何复制这个模式做其他网站

如果你将来想给其他项目做类似的网站，步骤如下：

### 第 1 步：创建 GitHub 账号
- 访问 github.com 注册（免费）

### 第 2 步：创建新仓库（Repository）
- 点 "New repository"
- 名字随意，比如 `my-new-website`
- 选 Public（免费的 GitHub Pages 需要公开仓库）

### 第 3 步：开启 GitHub Pages
- 仓库 Settings → Pages → Source 选 `main` 分支
- 保存后会分配一个 `你的用户名.github.io/仓库名` 的地址

### 第 4 步：买域名（可选，$10-15/年）
- 在 Cloudflare 注册账号
- 搜索想要的域名，购买
- 价格透明，`.com` 大概每年 $10-12

### 第 5 步：配置 DNS
- 在 Cloudflare DNS 设置里添加：
  - `CNAME` 记录：`@` → `你的用户名.github.io`
  - `CNAME` 记录：`www` → `你的用户名.github.io`
- 在仓库里创建 `CNAME` 文件，内容写域名

### 第 6 步：开始建网站
- 创建 `index.html`（首页）
- 用 AI（Claude）帮你写代码和设计
- 每次修改后 `git push` 上传

### 第 7 步：配置 SEO
- 创建 `sitemap.xml` 和 `robots.txt`
- 注册 Google Search Console
- 添加 Google Analytics 4

### 总费用
- 域名：$10-15/年（约 1500 日元/年）
- 托管：$0（GitHub Pages 免费）
- CDN：$0（Cloudflare 免费）
- SSL：$0（自动配置）
- **总计：每年只花域名钱！**

### 更新网站内容的流程
```
1. 打开管理后台 https://fuluckpet.com/admin/
2. 编辑猫咪信息
3. 如果需要更新前台页面：
   a. 在后台导出 HTML 代码
   b. 用 AI 帮你粘贴到对应的 HTML 文件
   c. git add → git commit → git push
4. 等 1-2 分钟确认更新
```

```
1. 访问 analytics.google.com
2. 登录你的 Google 账号
3. 选择 fuluckpet.com 的资源
4. "实时"看当前在线访客
5. "报告"看历史数据（用户数、页面浏览量等）

```
1. 访问 search.google.com/search-console

3. "效果"看搜索展示和点击数据

## 10. 常用术语表

| 术语 | 简单解释 |
|------|----------|
| **HTML** | 网页的骨架/内容文件，浏览器读它来显示网页 |
| **CSS** | 网页的化妆师，控制颜色、字体、布局 |
| **JS (JavaScript)** | 网页的大脑，控制点击、动画、交互 |
| **DOM** | 浏览器把 HTML 变成的"树形结构"，JS 通过它操控页面 |
| **Git** | 代码的时光机/版本管理工具 |
| **GitHub** | 存放代码的云端仓库（类似代码界的百度网盘） |
| **Repository (仓库)** | 一个项目的所有文件的集合 |
| **Commit (提交)** | 给代码拍一张快照，记录修改 |
| **Push (推送)** | 把本地的修改上传到 GitHub |
| **Pull (拉取)** | 把 GitHub 上的最新版本下载到本地 |
| **DNS** | 域名系统，互联网的电话簿（名字→IP 地址） |
| **CDN** | 内容分发网络，把网站缓存到全球各地的服务器 |
| **SSL/HTTPS** | 加密连接，网址前面的小锁，保护数据安全 |
| **Domain (域名)** | 网站地址，如 fuluckpet.com |
| **SEO** | 搜索引擎优化，让 Google 更容易找到你 |
| **OGP** | Open Graph Protocol，控制社交媒体分享预览的标签 |
| **Sitemap** | 网站地图，告诉 Google 你有哪些页面 |
| **robots.txt** | 告诉搜索引擎爬虫哪里可以去、哪里不能去 |
| **localStorage** | 浏览器里的本地存储空间，数据存在你的电脑里 |
| **sessionStorage** | 跟 localStorage 类似，但关闭浏览器就清空 |
| **i18n** | 国际化/多语言，internationalization 的缩写 |
| **JSON** | 一种数据格式，像结构化的文本文件 |
| **API** | 程序之间对话的接口（像餐厅的服务窗口） |
| **静态网站** | 所有内容提前写好，直接送给浏览器（快、简单） |
| **动态网站** | 每次访问时服务器现场生成内容（慢、复杂、但功能多） |
| **Cloudflare** | 一家提供 CDN、DNS、安全防护的公司（免费套餐够用） |
| **GitHub Pages** | GitHub 的免费网站托管服务 |
| **CNAME** | 一种 DNS 记录类型，把一个域名指向另一个域名 |
| **JSON-LD** | 帮搜索引擎理解网页内容的结构化数据格式 |
| **GA4** | Google Analytics 4，谷歌的网站流量统计工具 |
